## Bugs found and fixed during e2e

### Bug 1: validate-inventory.sh — FLOW category full-description regex mismatch

**Symptom:** inventory row `F-01 | FLOW | UC-01 user login | UC-01 | PENDING` was flagged as uncovered even though ARCHITECTURE.md contained a sequence diagram titled "Sequence Diagram - Login (UC-01)".

**Root cause:** the validator grepped for the full description `\bUC-01 user login\b` as a regex, requiring the exact literal phrase. ARCHITECTURE.md used the more natural "Login (UC-01)" form — valid evidence of coverage but didn't match.

**Fix:** extract the UC-NN / FL-NN / SC-NN stem from the description and grep for that as a fallback. See `scripts/validators/validate-inventory.sh:104-113`.

### Bug 2: validate-completion-manifest.sh — em-dash separator not detected

**Symptom:** manifest ending with "test-engineer done — auth module tests passing" was flagged as missing a completion phrase, even though the em-dash is the canonical separator.

**Root cause:** the regex `[---]` (intended to mean "hyphen or em-dash") collapsed to a character class of just `-` after em-dash-to-ASCII stripping in a previous fix. A perl fallback was added but used `\xE2` as a code point (not a byte), which doesn't match multi-byte UTF-8 under default perl.

**Fix:** build the em-dash literal via `printf '\xE2\x80\x94'` and splice into the grep alternation under `LC_ALL=C` so it matches as three bytes. See `scripts/validators/validate-completion-manifest.sh:70-76`.
